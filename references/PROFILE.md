# Demo 用户画像（PROFILE.md）

> 用途：`Scorer`、`Value-Mapper` 和 `Outlook-Curator` 会读取这个文件来理解读者是谁、什么内容重要、日报应如何判断价值。生产使用前必须替换这个 demo 画像。

## 1. 读者

demo 读者是一位通用知识工作者，希望从公开信源中获得可用于判断、研究、沟通或决策的每日简报。

### 1.1 背景

- 工作场景会受到公开新闻、专家分析、技术更新和长文评论影响。
- 需要简洁但可用于判断的信息，而不是通用新闻复述。
- 尚未提供某个特定领域的真实画像。

### 1.2 当前关注

- 理解过去 24 小时或配置时间窗口内的重要变化。
- 识别可执行动作、风险信号、市场 / 语境变化、技术影响或有用框架。
- 沉淀可复用笔记，用于研究、决策、内部 brief、写作或讨论。

### 1.3 偏好

- 偏好具体主张、数字、可观察变化和一手案例。
- 奖励非显而易见的信号和反共识观点。
- 降权浅层发布稿、模糊 hype、SEO filler 和不可读摘要。

## 2. 目标受众

demo 目标受众是通用专业读者，例如同事、stakeholder、读者、客户、学生、研究者、管理者或社区成员。

他们关心：

- 发生了什么变化，为什么重要；
- 哪些做法可以尝试、复制、避免或持续观察；
- 新信息如何改变成本、风险、行为、系统、角色、政策、市场或时机。

## 3. 输出定位

### 3.1 一句话定位

> 把公开信源信号转成可读、可用、可降噪的中文洞察日报。

### 3.2 内容标签

`public_signals` · `decision_support` · `risk_awareness` · `research_notes` · `context_shifts`

### 3.3 风格

- 具体、克制、基于证据。
- 当 `config/report_config.json` 设置 `output_language = "zh"` 时，将英文信源转成中文日报内容。
- 翻译会降低精确度的术语可保留英文。
- 不要写得像读者拥有某个私人用户的背景。

## 4. 使用场景

1. **读者学习**：更新读者的理解、决策、工作流或研究地图。
2. **可复用材料**：当用户需要时，沉淀可用于笔记、brief、文章或讨论的问题角度。

读者视角规则在 `references/perspectives/reader.md`。目标受众视角规则在 `references/perspectives/audience.md`。内容支柱是高级可选功能，不属于开源版默认日报。
